### Задание. Необходимо определить требуемые ресурсы
Известно, что проекту нужны база данных, система кеширования, а само приложение состоит из бекенда и фронтенда. Опишите, какие ресурсы нужны, если известно:

1. Необходимо упаковать приложение в чарт для деплоя в разные окружения. 
2. База данных должна быть отказоустойчивой. Потребляет 4 ГБ ОЗУ в работе, 1 ядро. 3 копии. 
3. Кеш должен быть отказоустойчивый. Потребляет 4 ГБ ОЗУ в работе, 1 ядро. 3 копии. 
4. Фронтенд обрабатывает внешние запросы быстро, отдавая статику. Потребляет не более 50 МБ ОЗУ на каждый экземпляр, 0.2 ядра. 5 копий. 
5. Бекенд потребляет 600 МБ ОЗУ и по 1 ядру на копию. 10 копий.

Ответ:
В начале подсчитываю сумарные ресурсы приложения на Поды:
## 1. Суммарные ресурсы для База данных - 12Gb ОЗУ, 3 ядра
   Суммарные ресурсы для Кеш - 12Gb ОЗУ, 3 ядра
   Суммарные ресурсы для Фронтенд - 250Mb ОЗУ, 1 ядро
   Суммарные ресурсы для Бекенд - 6Gb ОЗУ, 10 ядер

   Суммарные ресурсы всего приложения:
   ОЗУ = 12 + 12 + 0.25 + 6 = 30.25Gb
   CPU = 3 + 3 + 1 + 10 = 17 vCPU

## 2. Служебные ресурсы ноды
   CPU и ОЗУ распределяются между: ОС, Kubelet, Подами, Eviction threshold.
   На каждой ноде резервируются доп.ресурсы:
   OC + Kubelet занимают ~ 0.7% на CPU и 1.8Gb ОЗУ
   Eviction threshold резервирует около 100Mb.

   Итого служебные ресурсы на ноду:
   0.7 vCPU и 1.9Gb RAM
   
   Получается, итоговая формула:
   capacity ноды = общие ресурсы ноды – служебные ресурсы – eviction-threshold.

## 3. Для расчета считаем, что каждая нода имеет 8 vCPU и 16Gb ОЗУ, тогда:
   CPU для подов на одной ноде = 8 - 0.7 = 7.3 vCPU
   RAM для подов на одной ноде = 16Gb - 1.9Gb = 14.1Gb ОЗУ
   
   Таким образом, с учётом служебных ресурсов на одной ноде под пользовательские поды остаётся примерно 7.3 vCPU и 14.1Gb ОЗУ.
   Требования приложения: 17 vCPU и 30.25Gb ОЗУ

## 4. Текущее число нод (без учета отказоустойчивости):
    CPU = 17 (Суммарные ресурсы приложения CPU) / 7.3 (CPU для подов на одной ноде) = 3 (округляем в сторону большего числа)
    RAM = 30.25 (Суммарные ресурсы приложения ОЗУ) / 14.1 (RAM для подов на одной ноде) = 3 (округляем в сторону большего числа)
    Итого: Берем 3 ноды (без отказоустойчивости)

## 5. Для отказоустойчивости добавляем еще одну ноду:
   3 + 1 = 4
   
   При отказе одной ноды остаётся 3 ноды с суммарными ресурсами:
   21.9 vCPU и 42.3Gb ОЗУ, что превышает требования приложения (17 vCPU и 30.25Gb ОЗУ).

## 6. Итоговая конфигурация:
    4 ноды по 16Gb ОЗУ и 8 vCPU каждая
    Этого достаточно для размещения всех подов приложения и выдерживания отказа одной ноды.
